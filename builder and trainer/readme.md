# Builder and Trainer

The **Builder and Trainer** module contains the complete data preparation and model training pipeline used by **Guardian Vision**. It provides the tools required to preprocess the NTU RGB+D dataset, generate the training datasets, and train the custom Multi-Stream CTR-GCN model for human action recognition.

> **Note**
>
> The **NTU RGB+D 60 & 120** datasets are **not included** in this repository. Users must request and obtain the dataset directly from the official NTU RGB+D website before using the scripts provided in this folder.

---

# Overview

This module consists of three primary components:

1. **Dataset Builder**
2. **Multi-Stream CTR-GCN Model**
3. **Model Trainer**

Together, these components prepare the dataset, train the action recognition model, and generate the files required by Guardian Vision during inference.

Before using the Builder and Trainer pipeline, obtain the **NTU RGB+D 60 & 120 Action Recognition Dataset** from the official website.

Official Dataset:

https://rose1.ntu.edu.sg/dataset/actionRecognition/

After approval, download and extract the dataset.

> **Important**
>
> Preserve the original NTU RGB+D filenames (e.g., `S001C001P001R001A001`). These filenames contain metadata used during dataset preprocessing.

# 1. Dataset Builder

**Script**

```text
dataset_builder_combine_recommended.py
```

The dataset builder preprocesses the NTU RGB+D dataset and converts the extracted pose information into the format required by the Multi-Stream CTR-GCN model.

During preprocessing, the script performs the following operations:

- Reads the NTU RGB+D dataset
- Processes skeletal pose data
- Normalizes skeletal coordinates
- Generates normalization statistics
- Splits the dataset into training, validation, and testing subsets

---

## Dataset Split

The processed dataset is automatically divided into:

| Dataset | Ratio |
|---------|------:|
| Training | 80% |
| Validation | 10% |
| Testing | 10% |

---

## Generated Files

Running the dataset builder produces the following files:

```text
train.pkl
val.pkl
test.pkl
norm_stats_coco17_2d_T36_M1.pkl
```

### File Description

| File | Purpose |
|------|---------|
| `train.pkl` | Training dataset used by the model trainer. |
| `val.pkl` | Validation dataset used during training. |
| `test.pkl` | Testing dataset used for model evaluation. |
| `norm_stats_coco17_2d_T36_M1.pkl` | Normalization statistics required by Guardian Vision during inference. |

The generated **train**, **validation**, and **test** datasets are used directly by the training script.

The normalization statistics file should later be copied into:

```text
backend/assets/
```

---

## Running the Dataset Builder

```bash
python builder_trainer/dataset_builder_combine_recommended.py
```

# 2. Multi-Stream CTR-GCN Model

**Script**

```text
ctrgcn_model.py
```

Guardian Vision utilizes a custom implementation of a **Multi-Stream Channel-wise Topology Refinement Graph Convolutional Network (CTR-GCN)** for skeleton-based human action recognition.

The model processes three complementary skeletal representations:

- Joint Stream
- Bone Stream
- Motion Stream

Each stream independently learns spatial-temporal features before the extracted representations are fused to perform action classification.

The model is designed for:

- COCO-17 skeleton format
- Skeleton-based action recognition
- Efficient model training
- Real-time inference integration with Guardian Vision

---

# 3. Model Trainer

**Script**

```text
training_with_cctv_aug.py
```

The training script trains the Multi-Stream CTR-GCN model using the processed datasets generated by the dataset builder.

The training process utilizes:

- `train.pkl`
- `val.pkl`
- `test.pkl`

to train, validate, and evaluate the action recognition model.

After training is completed, the script generates:

```text
ctrgcn_bundle.pth
```

This trained model is required by Guardian Vision during inference.

Copy the generated model into:

```text
backend/assets/
```

---

## Running the Trainer

```bash
python builder_trainer/training_with_cctv_aug.py
```

# External Dependencies

## RTMPose

This project uses **RTMPose-S (COCO, 17 keypoints, 256×192 input)** for human pose estimation.

Download the corresponding pretrained model from the official MMPose project:

- https://github.com/open-mmlab/mmpose

Rename the downloaded checkpoint to:

```text
rtmpose.pth
```

and place it in:

```text
backend/assets/
```

---

## YOLOv8

This project uses **YOLOv8n** for human detection.

Download the pretrained model from:

- https://github.com/ultralytics/ultralytics

Place the downloaded checkpoint into:

```text
backend/assets/
```
